Tutorial - Static Website on GitHub Pages

To submit the final project report, you will create a single-page static website that is effectively a Jupyter notebook rendered to HTML, and host it using GitHub Pages.

The final product should be viewable at a public URL and require no server or backend.

1. Prepare the Notebook

  1. Create or finalize a Jupyter notebook (.ipynb).

  2. Make sure it:

    • Runs top-to-bottom without errors
    • Has clear section headers (Markdown cells)
    • Includes all figures inline (no external display windows)

  3. Restart the kernel and Run All to ensure reproducibility.

  4. Save the notebook.

Recommended:

  • Use Markdown headings (#, ##) to structure the page.
  • Avoid interactive widgets that require a live kernel.

2. Convert Notebook to HTML

Download the notebook. If you have Jupyter installed on your computer, you can run:

jupyter nbconvert --to html your_notebook.ipynb

This creates:

your_notebook.html

Optional but useful flags:

jupyter nbconvert --to html --embed-images your_notebook.ipynb

This ensures figures are embedded directly in the HTML file.

If you don't have Jupyter installed, just download the notebook from Colab, and import it back into another Colab session. Then run the command above.

In[2]:
!jupyter nbconvert --to html --embed-images final_project_report_submission.ipynb

[NbConvertApp] Converting notebook final_project_report_submission.ipynb to html
[NbConvertApp] Writing 280703 bytes to final_project_report_submission.html

3. Create a GitHub Repository

  1. Go to GitHub and create a new repository or use your project repo.
  2. Repository settings:
    • Public (required for free GitHub Pages)
    • Any name is fine (e.g. project-website)
  3. Rename your_notebook.html index.html
  4. Upload index.html to the repo
  • GitHub Pages always serves index.html by default.

4. Enable GitHub Pages

  1. In the repository, go to Settings Pages

  2. Under Source:

    • Branch: main
    • Folder: / (root)

  3. Save.

GitHub will display a URL like:

https://username.github.io/repository-name/

Deployment may take 12 minutes.

5. Verify the Website

  1. Open the GitHub Pages URL in a private/incognito window.

  2. Confirm:

    • Page loads without errors
    • Figures and equations render correctly
    • Text is readable and well-formatted

This URL is what you submit.
